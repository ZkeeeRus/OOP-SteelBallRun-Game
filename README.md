> Двухмерная гоночная аркада для двоих игроков. Динамическая камера,
> система бонусов, процедурный звук и чистая архитектура на OpenGL.

## О проекте

Проект представляет собой полноценный раннер. Игроки управляют двумя персонажами, соревнуясь на дистанции. Камера автоматически адаптируется. При близком расположении игроков выводится общий экран. При отдалении экран разделяется на две независимые зоны.

## Ключевые особенности

-   Умная камера. Плавный переход между общим и разделённым экраном на основе дистанции между игроками
-   Система бонусов. Конфигурация через `bonuses.json`. Поддержка множителей скорости, ускорения и временной неуязвимости. Возможность добавлять новые бонусы и редактировать текущие без необходимости перекомпиляции проекта
-   Звук. Фоновая музыка с зацикливанием, кэширование SFX, рандомизация питча для шагов, потоковый микшер без задержек
-   Тестирование. Покрытие базовой механики через xUnit

## Управление

- Игрок 1 (Джайро): `W` - прыжок
- Игрок 2 (Джонни): `Стрелочка вверх` - прыжок 
- `Esc` - закрытие игрового окна
- `F1`, `F2`, `F3` - встроенные дебаггеры

## Технологический стек

-   Язык: C# / .NET 8+
-   Графика: OpenTK
-   Аудио: NAudio
-   UI: Windows Forms (базовое окно, кастомный рендеринг поверх GLControl)

## Сборка и запуск

1.  Убедитесь, что папка `Content` находится рядом с исполняемым файлом и содержит:
    -   `Images/` (текстуры игроков, фона, препятствий, бонусов)
    -   `Audio/Music/` и `Audio/SFX/`
    -   `Bonuses/bonuses.json`
    -   `Shaders/`
2.  Сборка:
```bash
dotnet build --configuration Release
```
3. Запуск:
```bash
dotnet run
```
## Структура проекта
```
SBR-Game/
├── Core/          # GameLogic, PlayerState, GameObjectFactory
├── Gameplay/      # Player, Obstacles, WarningEffect
├── Bonuses/       # Registry, Modifiers, Pickups
├── Rendering/     # OpenGL рендерер, шейдеры, фоны, UI
├── Audio/         # NAudio микшер, кэш звуков, провайдеры
└── States/        # Управление фазами игры (MainMenu, Playing, Finish)
```

_Проект создан в учебных целях. Архитектура оптимизирована под 60 FPS, разделение логики и рендеринга, а также поддержка расширяемого контента через JSON и файловую систему._
